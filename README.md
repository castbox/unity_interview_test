# Guru团队上机测试题

你好，

欢迎参加 **Guru Game Unity** 团队上机测试题

请在测试开始前认真阅读以下说明:

## 注意事项

- 测试题库将由 **HR** 或 **面试官** 直接发送给你本人, 发送的形式将是一份 `github` 仓库的分支拉取说明。
- Unity3D 的版本推荐使用以下版本, 其他版本若校验时出现插件问题则视为不通过。
  - [x] 2022.3.62f2 ( Test 默认版本 )
  - [ ] 6000.0.59f2 ( 可选 )
- 没有特殊说明, 默认的测试时长控制在 **60分钟** 以内.
  - 阅读规则，不作计时
  - 下载和拉取 Repo 不作计时
  - 下载和安装 Unity 不作计时
  - 提交和上传测试文件不作计时
- 请按照指定的仓库链接下载仓库到本地, 并切换到说明中指定的 **分支**
- 请使用以下规范创建你的 < **个人文件夹** >：
  - 使用你的姓名的 <u>全汉语拼音小写字母</u>, 拼接 `_{年}{月}{日}{时}` 的方式, 于Unity工程 Assets 目录下创建你的个人文件夹, 例如:
      ```
      面试者姓名: 王小刚
      开始时间: 北京时间 2025年1月12日 下午 2:07

      则创建个人文件夹名称为:  
      wangxiaogang_25011214
    
      创建后的目录结构如下:
      Assets/wangxiaogang_25011214
      ```
- 请于 < **个人文件夹** > 内创建合理的文件名称, 用于区分各类资源, 你所创建的脚本，预制体，图片或者 Shader 等文件，最终都 **必须** 放置在此目录内。
- 你可以使用项目内 `Assets/StreamingAssets` 或 `Assets/Resources` 来创建和加载相关的资源文件
- 试题的前置和环境文件部署在 `Assets/__quest` 路径内, 请勿修改此路径内部的文件.
- 可使用其他三方插件库，但原则上插件库不能替代你完成参与测试的核心逻辑。功能相关的核心逻辑必须独立完成
- 项目内已经为你准备了以下的第三方库, 方便你直接调用。
   - DoTween: 简单的补间动画可用
   - JsonConvert: Json 格式解析可用
 - 请确保 最后完成的程序满足了 **所有的功能**，没有遗漏，包括提交和格式的需求。请充分的自测
 - 请确保 **代码执行效率是高的, 没有冗余和内存溢出的风险**
 - 完成后请参考以下建议将项目打包为 `.zip` 文件, 在指定的时间内, 发送给HR或者面试官
   - 要打包的文件夹仅限于 `Assets, ProjectSettings, Packages` 三个文件夹的内容， **其他文件夹请勿提交**， 节约上传和下载的时间
   - 提交前请针对考察要求充分自测，避免出现功能缺失或者低级错误


## 测试反馈
 - 你在此次测试中发现了问题或有任何不满意的地方, 欢迎在 <**个人文件夹**>/`FEEDBACK.md` 文档中积极的书写意见和建议.
 - 根据反馈会视其内容和价值增加一定的分数
